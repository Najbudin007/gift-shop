Recap: Basic Elements

Let's review the core building blocks we learned.

<p> Paragraph

Used for blocks of text. It automatically adds spacing before and after.

<a> Anchor (Links)

The href attribute is mandatory.

<a href="google.com">Click</a>
<img> Images

Self-closing tag. Key attributes: src (source) and alt (alternative text).

<img src="pic.jpg" alt="A cat">
Note: Attributes like class, id, and style can be used on almost all elements.

Theory: Character Sets

How does the browser know how to display text?

To display an HTML page correctly, a web browser must know the character set (encoding) to use.
<meta charset="UTF-8">
The Evolution:

The first standard. Supported only 128 characters (English letters, numbers, and basic symbols). No foreign accents or emojis.

Expanded to 256 characters. This allowed for Western European languages (like French, German) but still couldn't handle Asian languages or complex symbols.

Why we use UTF-8

UTF-8 (Unicode Transformation Format) is the modern standard for HTML5.

HTML Layouts

Websites are not just one long column of text. They have structure.

Semantic Elements

HTML5 introduced tags that describe what the content is, rather than just how it looks.

  • <header>: Top area (Logo, Title)
  • <nav>: Menu links
  • <section>: A specific section of content
  • <article>: Independent content (blog post)
  • <footer>: Bottom area (Copyright, contact)

How to Create Layouts

We know the tags (header, footer), but how do we arrange them side-by-side?

1. Floats
The "Old Way". Using float: left. Hard to manage and requires "clearing" floats. Not recommended for main layouts anymore.
2. Flexbox
Great for 1-dimensional layouts (rows OR columns). Excellent for aligning items inside a container.
3. CSS Grid
The most powerful system. Handles 2-dimensional layouts (rows AND columns) simultaneously.
4. Frameworks
Tools like Bootstrap or W3.CSS use Flexbox/Grid under the hood to make layouts easy (like this presentation!).
